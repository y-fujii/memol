memol

Memol is a music notation language focused on MIDI. It is well-structured, score friendly and easy to read, compared with another one.

1. Features

1.1. Core notation

Memol specify the note duration not by number but by dividing the unit duration.

Core notation of Memol has only two structures: "(...)" and "[...]". "[...]" represents group notes. It divide its duration equally into child notes. "(...)" represents chord. Its child notes are put at the same time. These can be nested each other.

Here is a example coded in usual notation:

a4 b4 < c4. d16 d16 c8. > b16 a16 < e0 g0 b8. f8 e4. b2 a2

coded in Memol:

{ [a B] [C. [D D]] | [c. b] [a E.] [F (e g b)^] / | b a }
{ a B 3C [D D] | [3c b] [a 3E] [F (e g b)^] / | b a }

As you can see, Memol looks much prettier.

1.2. Relative octave

Memol has a mechanism to avoid annoying octave changing such as "c<b>c<b". If you write a note in upper case, it has higher pitch than previous one within a octave. If in lower case, it has lower pitch within a octave.

1.3. Separation of chord, backing form, expression

Memol aims to separate some musical structure such as chord, backing form, expression (control change, gate time, etc..)
